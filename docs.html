About the Queryable API

The Queryable API is Microbot's answer to inconsistent game entity querying. It provides a type-safe, chainable way to filter and find NPCs, players, ground items, and tile objects with minimal boilerplate code. It's designed to make your code more readable, maintainable, and less error-prone.

Why Use Queryable API?

Old Way ‚ùå

Legacy 
// Using legacy Rs2Npc API
// Simple but limited
NPC banker = Rs2Npc
    .getNpc("Banker");

// More complex queries get verbose
NPC cow = Rs2Npc.getNpcs()
    .stream()
    .filter(npc -> npc.getName()
        .equals("Cow"))
    .filter(npc -> npc
        .getInteracting() == null)
    .min(Comparator.comparingInt(
        npc -> npc.getWorldLocation()
            .distanceTo(Rs2Player
                .getWorldLocation())))
    .orElse(null);

New Way ‚úÖ

Queryable 
// Clean, readable, and concise
Rs2NpcModel banker =
    new Rs2NpcQueryable()
        .withName("Banker")
        .nearest();

// Complex queries stay readable
Rs2NpcModel cow =
    new Rs2NpcQueryable()
        .withName("Cow")
        .where(npc ->
            !npc.isInteracting())
        .nearest();

Core Concepts

1. Fluent Chaining

Methods return the queryable itself, allowing you to chain multiple operations:

new Rs2NpcQueryable()
    .withName("Guard")           // Filter by name
    .where(npc -> !npc.isInteracting())  // Custom filter
    .within(15)                  // Within 15 tiles
    .nearest();                  // Get nearest match

2. Lazy Evaluation

Queries are only executed when you call a terminal operation:

// No execution yet - just building the query
Rs2NpcQueryable query = new Rs2NpcQueryable()
    .withName("Guard")
    .within(10);

// NOW it executes
Rs2NpcModel guard = query.nearest();  // Terminal operation

3. Type Safety

All operations are type-checked at compile time, preventing runtime errors:

// Compile-time safety
Rs2NpcModel npc = new Rs2NpcQueryable()
    .withName("Banker")
    .nearest();  // Returns Rs2NpcModel, not generic Object

Getting Started

Every query follows this simple pattern:

// 1. Create queryable
new Rs2NpcQueryable()

// 2. Add filters (optional, chainable)
    .withName("name")
    .where(entity -> condition)
    .within(distance)

// 3. Execute with terminal operation
    .nearest();  // or first(), toList(), etc.

Rs2NpcQueryable - NPC Queries

Import: net.runelite.client.plugins.microbot.api.npc.Rs2NpcQueryable

Basic Examples

Find nearest banker:

Rs2NpcModel banker = new Rs2NpcQueryable()
    .withName("Banker")
    .nearest();

Find all guards within 10 tiles:

List<Rs2NpcModel> guards = new Rs2NpcQueryable()
    .withName("Guard")
    .within(10)
    .toList();

Find nearest non-interacting cow:

Rs2NpcModel cow = new Rs2NpcQueryable()
    .withName("Cow")
    .where(npc -> !npc.isInteracting())
    .nearest();

Combat script example - find attackable enemy:

Rs2NpcModel enemy = new Rs2NpcQueryable()
    .withName("Goblin")
    .where(npc -> !npc.isInteracting())
    .where(npc -> npc.getHealthRatio() > 0)
    .nearest(10);

if (enemy != null && !Rs2Player.isInCombat()) {
    enemy.click("Attack");
    sleepUntil(() -> Rs2Player.isInCombat(), 2000);
}

Available Methods

  • npc.getName() - Get NPC name
  • npc.getId() - Get NPC ID
  • npc.isInteracting() - Check if NPC is interacting
  • npc.getHealthRatio() - Get health ratio (0-30)
  • npc.getAnimation() - Get current animation ID
  • npc.click("Action") - Interact with NPC

Rs2TileItemQueryable - Ground Items

Import: net.runelite.client.plugins.microbot.api.tileitem.Rs2TileItemQueryable

Basic Examples

Find nearest coins:

Rs2TileItemModel coins = new Rs2TileItemQueryable()
    .withName("Coins")
    .nearest();

Find valuable loot:

Rs2TileItemModel loot = new Rs2TileItemQueryable()
    .where(item -> item.getTotalGeValue() >= 5000)
    .where(Rs2TileItemModel::isLootAble)
    .nearest(10);

if (loot != null) {
    loot.pickup();
}

Find items about to despawn:

List<Rs2TileItemModel> despawning = new Rs2TileItemQueryable()
    .where(item -> item.willDespawnWithin(30))  // 30 seconds
    .where(item -> item.getTotalValue() > 1000)
    .toList();

Available Methods

  • item.getName() - Get item name
  • item.getQuantity() - Get stack size
  • item.getTotalValue() - Get total value
  • item.getTotalGeValue() - Get total GE value
  • item.isLootAble() - Check if lootable
  • item.pickup() - Pick up the item
  • item.willDespawnWithin(seconds) - Check despawn time

API Reference

Terminal Operations

These execute the query and return results:

nearest()

Returns the nearest entity to the player.

Rs2NpcModel npc = new Rs2NpcQueryable()
    .withName("Guard")
    .nearest();

nearest(int maxDistance)

Returns the nearest entity within max distance from player.

Rs2NpcModel npc = new Rs2NpcQueryable()
    .withName("Guard")
    .nearest(10);  // Within 10 tiles

first()

Returns the first matching entity (not necessarily nearest).

Rs2NpcModel npc = new Rs2NpcQueryable()
    .withName("Guard")
    .first();

toList()

Returns all matching entities as a list.

List<Rs2NpcModel> guards = new Rs2NpcQueryable()
    .withName("Guard")
    .toList();

Filter Operations

These filter entities and return the queryable for chaining:

where(Predicate<E> predicate)

Adds a custom filter using a lambda expression.

new Rs2NpcQueryable()
    .where(npc -> npc.getHealthRatio() > 0)
    .where(npc -> !npc.isInteracting())
    .nearest();

within(int distance)

Filters entities within distance from player.

new Rs2NpcQueryable()
    .withName("Guard")
    .within(10)
    .toList();

Common Patterns

Find by Multiple Names

Rs2NpcModel banker = new Rs2NpcQueryable()
    .withNames("Banker", "Bank clerk", "Bank assistant");

if (banker != null) {
    banker.click("Bank");
}

Complex Query with Multiple Filters

Rs2NpcModel target = new Rs2NpcQueryable()
    .withName("Goblin")
    .where(npc -> !npc.isInteracting())
    .where(npc -> npc.getHealthRatio() > 0)
    .where(npc -> npc.getAnimation() == -1)  // Not animating
    .within(10)
    .nearest();

Reusable Predicates

// Define predicates
Predicate<Rs2NpcModel> isAlive = npc -> npc.getHealthRatio() > 0;
Predicate<Rs2NpcModel> notBusy = npc -> !npc.isInteracting();
Predicate<Rs2NpcModel> notAnimating = npc -> npc.getAnimation() == -1;

// Use them
Rs2NpcModel target = new Rs2NpcQueryable()
    .withName("Cow")
    .where(isAlive)
    .where(notBusy)
    .where(notAnimating)
    .nearest();

Performance Tips

‚úÖ DO

  • ‚Ä¢ Use specific filters early
  • ‚Ä¢ Limit search radius with within()
  • ‚Ä¢ Cache results when possible
  • ‚Ä¢ Use method references
  • ‚Ä¢ Add reasonable delays between queries

‚ùå DON'T

  • ‚Ä¢ Query in tight loops without delays
  • ‚Ä¢ Use expensive operations in filters
  • ‚Ä¢ Create unnecessary intermediate lists
  • ‚Ä¢ Forget null checks
  • ‚Ä¢ Query without distance limits

Good Example

// Good - filters by name first, limits radius
new Rs2NpcQueryable()
    .withName("Guard")
    .where(npc -> !npc.isInteracting())
    .within(10)
    .nearest();

Migration Guide

Migrating from the legacy API is straightforward. Here are common conversions:

NPCs

Legacy Rs2Npc:

// Simple query
NPC npc = Rs2Npc
  .getNpc("Banker");

// With filter
NPC npc = Rs2Npc
  .getNearestNpc(
    npc -> !npc.isInteracting());

Queryable:

// Simple query
Rs2NpcModel npc =
  new Rs2NpcQueryable()
    .withName("Banker");

// With filter
Rs2NpcModel npc =
  new Rs2NpcQueryable()
    .where(npc ->
      !npc.isInteracting())
    .nearest();

Ground Items

Legacy Rs2GroundItem:

TileItem item =
  Rs2GroundItem
    .getGroundItem("Coins");

Queryable:

Rs2TileItemModel item =
  new Rs2TileItemQueryable()
    .withName("Coins")
    .nearest();

Tile Objects

Legacy Rs2GameObject:

TileObject tree =
  Rs2GameObject
    .findObject("Tree");

Queryable:

Rs2TileObjectModel tree =
  new Rs2TileObjectQueryable()
    .withName("Tree")
    .nearest();
